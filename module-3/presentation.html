3.2 Azure Container Registry Fundamentals

What is ACR?

Azure Container Registry is a private Docker registry service that stores and manages container images and related OCI artifacts.

Key Features:

  • Private registry: Secure storage for your container images
  • Geo-replication: Replicate images across Azure regions (Premium)
  • ACR Tasks: Automated image builds and patching
  • Webhooks: Trigger actions on push/delete events
  • Security scanning: Vulnerability assessment with Defender
  • OCI artifacts: Store Helm charts, SBOMs, configs
  • Integration: Native support for AKS, Container Apps, App Service

Provisioning ACR with Azure CLI

Create Your First Registry

# Create resource group
az group create \
  --name rg-containers \
  --location eastus

# Create ACR (Premium tier)
az acr create \
  --resource-group rg-containers \
  --name myuniqueregistry \
  --sku Premium \
  --location eastus

# Enable admin credentials (dev/test only)
az acr update \
  --name myuniqueregistry \
  --admin-enabled true

# Get credentials
az acr credential show \
  --name myuniqueregistry \
  --output table

# Login to ACR
az acr login --name myuniqueregistry

Note: Registry name must be globally unique and contain only alphanumeric characters.

3.3 Working with Container Images

Push Images to ACR

Step 1: Tag Your Image

# Tag for ACR
docker tag mywebapi:1.0 myuniqueregistry.azurecr.io/mywebapi:1.0
docker tag mywebapi:1.0 myuniqueregistry.azurecr.io/mywebapi:latest

Step 2: Push to ACR

# Login first
az acr login --name myuniqueregistry

# Push images
docker push myuniqueregistry.azurecr.io/mywebapi:1.0
docker push myuniqueregistry.azurecr.io/mywebapi:latest

Step 3: Verify in ACR

# List repositories
az acr repository list --name myuniqueregistry --output table

# Show tags for repository
az acr repository show-tags \
  --name myuniqueregistry \
  --repository mywebapi \
  --output table

Managing Image Lifecycle

Delete, Purge, and Maintain

Delete Specific Tag:

# Delete tag
az acr repository delete \
  --name myuniqueregistry \
  --image mywebapi:old-version \
  --yes

Purge Old Images (Premium):

# Purge images older than 30 days
az acr run \
  --registry myuniqueregistry \
  --cmd "acr purge --filter 'mywebapi:.*' --ago 30d --untagged" \
  /dev/null

Retention Policy:

# Set retention policy (Premium)
az acr config retention update \
  --registry myuniqueregistry \
  --status enabled \
  --days 7 \
  --type UntaggedManifests

3.4 ORAS CLI for OCI Artifacts

Beyond Container Images

What are OCI Artifacts?

OCI (Open Container Initiative) artifacts extend the registry standard to store any type of content, not just container images.

Common Use Cases:

  • Helm Charts: Kubernetes application packages
  • Configuration Files: Application configs, policies
  • Terraform Modules: Infrastructure as Code
  • ML Models: Machine learning models and datasets
  • SBOMs: Software Bill of Materials
  • Signatures: Image signatures and attestations

ORAS vs Docker: ORAS is designed specifically for OCI artifacts, while Docker CLI focuses on container images.

Pushing Artifacts with ORAS

Store Configuration Files

Setup:

# Login to ACR
az acr login --name myuniqueregistry
export REGISTRY=myuniqueregistry.azurecr.io

Push Single File:

# Create config file
echo '{"setting": "value"}' > config.json

# Push as artifact
oras push $REGISTRY/configs/myapp:v1 \
  --artifact-type application/vnd.mycompany.config.v1+json \
  config.json:application/json

Push Multiple Files:

# Push deployment package
oras push $REGISTRY/artifacts/deployment:v1 \
  --artifact-type application/vnd.mycompany.deployment.v1+tar \
  ./deployment.yaml:application/yaml \
  ./values.yaml:application/yaml \
  ./README.md:text/markdown

Storing Helm Charts in ACR

Kubernetes Application Packages

Push Helm Chart:

# Package chart
helm package ./mychart

# Login to ACR
az acr login --name myuniqueregistry

# Push chart as OCI artifact
helm push mychart-1.0.0.tgz oci://myuniqueregistry.azurecr.io/helm

Pull and Install:

# Pull chart
helm pull oci://myuniqueregistry.azurecr.io/helm/mychart --version 1.0.0

# Install directly from ACR
helm install myapp \
  oci://myuniqueregistry.azurecr.io/helm/mychart \
  --version 1.0.0

List Charts:

# List Helm charts in ACR
az acr repository list \
  --name myuniqueregistry \
  --output table | grep helm/

Attaching SBOMs and Signatures

Supply Chain Security

Generate and Attach SBOM:

# Generate SBOM
docker sbom mywebapi:1.0 \
  --format spdx-json \
  --output sbom.spdx.json

# Attach to image in ACR
export REGISTRY=myuniqueregistry.azurecr.io
oras attach $REGISTRY/mywebapi:1.0 \
  --artifact-type application/vnd.example.sbom.v1+json \
  sbom.spdx.json:application/json

Discover Attachments:

# View all artifacts attached to image
oras discover $REGISTRY/mywebapi:1.0

# View with tree format
oras discover $REGISTRY/mywebapi:1.0 --format tree

3.5 ACR Security and Access Control

Authentication Methods

Authentication Options:

  • Admin account: Simple but less secure (dev/test only)
  • Service principal: Application identity with RBAC
  • Managed identity: Azure-managed identity (recommended)
  • Azure AD user: Individual user authentication
  • Repository-scoped tokens: Fine-grained access (Premium)

Create Service Principal:

# Create SP with acrpull role
az ad sp create-for-rbac \
  --name acr-service-principal \
  --scopes /subscriptions/{sub-id}/resourceGroups/rg-containers/providers/Microsoft.ContainerRegistry/registries/myuniqueregistry \
  --role acrpull

Network Security

Private Endpoints and Firewalls

Disable Public Access (Premium):

# Disable public network access
az acr update \
  --name myuniqueregistry \
  --public-network-enabled false

Configure Firewall Rules:

# Add IP address to allow list
az acr network-rule add \
  --name myuniqueregistry \
  --ip-address 40.76.54.131

# Add IP range
az acr network-rule add \
  --name myuniqueregistry \
  --ip-address 40.76.54.0/24

Create Private Endpoint:

# Create private endpoint
az network private-endpoint create \
  --resource-group rg-containers \
  --name myacr-private-endpoint \
  --vnet-name myvnet \
  --subnet mysubnet \
  --private-connection-resource-id $(az acr show --name myuniqueregistry --query id -o tsv) \
  --group-id registry \
  --connection-name myacr-connection

Image Scanning and Defender

Vulnerability Assessment

Enable Microsoft Defender for Containers:

# Enable Defender
az security pricing create \
  --name Containers \
  --tier Standard

# Get vulnerability assessment
az security assessment list \
  --resource-id "/subscriptions/{sub-id}/resourceGroups/rg-containers/providers/Microsoft.ContainerRegistry/registries/myuniqueregistry"

Scan with ACR Tasks:

# Create scan task
az acr task create \
  --registry myuniqueregistry \
  --name security-scan \
  --context /dev/null \
  --cmd "mcr.microsoft.com/azure-cli trivy image {{.Run.Registry}}/{{.Run.Repository}}:{{.Run.Tag}}" \
  --schedule "0 0 * * *"
