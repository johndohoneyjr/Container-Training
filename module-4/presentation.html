4.2 Azure Container Instances (ACI)

Fastest Way to Run Containers

Key Features:

  • Per-second billing: Pay only for what you use
  • Fast startup: Containers start in seconds
  • Hypervisor isolation: Secure container groups
  • Custom sizes: Flexible CPU and memory allocation
  • Persistent storage: Azure Files volume mounting
  • Public IP and DNS: Direct internet access

Deploy Container:

az container create \
  --resource-group rg-containers \
  --name myapi-aci \
  --image myregistry.azurecr.io/mywebapi:1.0 \
  --dns-name-label myapi-unique \
  --ports 8080

4.3 Azure Container Apps (ACA)

Serverless Containers for Microservices

Key Features:

  • Managed Kubernetes: Built on KEDA and Envoy
  • Auto-scaling: HTTP, queue, CPU, custom metrics
  • Revisions: Versioning and traffic splitting
  • Dapr integration: Built-in service-to-service calls
  • Scale to zero: Pay only when running
  • Built-in ingress: HTTPS with managed certificates

Create Environment:

az containerapp env create \
  --name myenv \
  --resource-group rg-containers \
  --location eastus

Deploying to Container Apps

Create and Configure App

Deploy Container App:

az containerapp create \
  --name mywebapi \
  --resource-group rg-containers \
  --environment myenv \
  --image myregistry.azurecr.io/mywebapi:1.0 \
  --target-port 8080 \
  --ingress external \
  --min-replicas 1 \
  --max-replicas 10

Configure Autoscaling:

az containerapp update \
  --name mywebapi \
  --resource-group rg-containers \
  --scale-rule-name http-rule \
  --scale-rule-type http \
  --scale-rule-http-concurrency 50

Get App URL:

az containerapp show \
  --name mywebapi \
  --resource-group rg-containers \
  --query properties.configuration.ingress.fqdn -o tsv

Blue-Green Deployments

Traffic Splitting Between Revisions

Revision Management:

  • Each deployment creates a new revision
  • Revisions are immutable and versioned
  • Split traffic across multiple revisions
  • Instant rollback to previous revision
  • A/B testing with traffic weights

Split Traffic:

# Route 80% to v1, 20% to v2
az containerapp ingress traffic set \
  --name mywebapi \
  --resource-group rg-containers \
  --revision-weight mywebapi--v1=80 mywebapi--v2=20

# Shift all traffic to v2
az containerapp ingress traffic set \
  --name mywebapi \
  --resource-group rg-containers \
  --revision-weight mywebapi--v2=100

Deploy Container to App Service

ACR Integration

Create Web App:

az webapp create \
  --resource-group rg-containers \
  --plan myappplan \
  --name mywebapi-app \
  --deployment-container-image-name myregistry.azurecr.io/mywebapi:1.0

Configure ACR Credentials:

az webapp config container set \
  --name mywebapi-app \
  --resource-group rg-containers \
  --docker-registry-server-url https://myregistry.azurecr.io \
  --docker-registry-server-user  \
  --docker-registry-server-password

Enable Continuous Deployment:

az webapp deployment container config \
  --name mywebapi-app \
  --resource-group rg-containers \
  --enable-cd true

4.5 Azure Kubernetes Service (AKS)

Enterprise-Grade Container Orchestration

Why AKS?

  • Full Kubernetes: Complete K8s API and ecosystem
  • Managed control plane: Microsoft manages masters
  • ACR integration: Seamless image pulling
  • Auto-scaling: HPA and cluster autoscaler
  • Advanced networking: Azure CNI, Calico policies
  • Security: Azure AD, RBAC, Key Vault integration

Prerequisites: Basic Kubernetes knowledge (pods, services, deployments)

Provisioning AKS Cluster

Create with ACR Integration

Create AKS Cluster:

az aks create \
  --resource-group rg-containers \
  --name myakscluster \
  --node-count 3 \
  --node-vm-size Standard_D2s_v3 \
  --enable-managed-identity \
  --attach-acr myregistry \
  --network-plugin azure \
  --enable-addons monitoring \
  --generate-ssh-keys

Get Credentials:

az aks get-credentials \
  --resource-group rg-containers \
  --name myakscluster

kubectl get nodes

Deploy Application to AKS

Kubernetes Manifests

Deployment Manifest:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: mywebapi
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mywebapi
  template:
    metadata:
      labels:
        app: mywebapi
    spec:
      containers:
      - name: mywebapi
        image: myregistry.azurecr.io/mywebapi:1.0
        ports:
        - containerPort: 8080

Apply Deployment:

kubectl apply -f deployment.yaml
kubectl get pods -l app=mywebapi

AKS Autoscaling

Horizontal Pod Autoscaler

Create HPA:

kubectl autoscale deployment mywebapi \
  --cpu-percent=70 \
  --min=2 \
  --max=20

kubectl get hpa mywebapi --watch

Enable Cluster Autoscaler:

az aks update \
  --resource-group rg-containers \
  --name myakscluster \
  --enable-cluster-autoscaler \
  --min-count 1 \
  --max-count 10

HPA: Scales pods based on CPU/memory/custom metrics
Cluster Autoscaler: Scales nodes based on pending pods

GitHub Actions for Containers

Build, Push, and Deploy

Workflow Example:

name: Deploy to AKS

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Azure Login
      uses: azure/login@v1
      with:
        creds: ${{ secrets.AZURE_CREDENTIALS }}
    
    - name: Build and push to ACR
      run: |
        az acr build \
          --registry myregistry \
          --image mywebapi:${{ github.sha }} \
          --file Dockerfile .
    
    - name: Deploy to AKS
      run: |
        az aks get-credentials -g rg-containers -n myakscluster
        kubectl set image deployment/mywebapi \
          mywebapi=myregistry.azurecr.io/mywebapi:${{ github.sha }}
