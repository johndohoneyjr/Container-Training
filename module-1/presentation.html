1.1 Introduction to Containers

What are Containers?

Containers are lightweight, standalone, executable packages that include everything needed to run a piece of software, including the code, runtime, system tools, libraries, and settings.

Key Characteristics:

  • Isolation: Process and resource isolation without full virtualization
  • Portability: Run consistently across different environments
  • Efficiency: Share OS kernel, minimal overhead
  • Fast startup: Boot in seconds, not minutes
  • Immutability: Same container image produces identical results

Container Runtime Environments & Standards

  • CRI-O: Lightweight alternative designed for Kubernetes

Industry standards for container formats and runtime:

  • Runtime Spec: Defines container execution environment

1.2 Container Ecosystem

Docker Architecture

  • Docker Engine: Core component that creates and runs containers
  • Docker CLI: Command-line interface for interacting with Docker
  • Docker Daemon (dockerd): Background service managing containers
  • Docker Desktop: GUI application for Mac/Windows

Key Components

  • Images: Read-only templates with application code and dependencies
  • Layers: Images built from layered filesystems for efficiency
  • Containers: Running instances of images
  • Volumes: Persistent data storage
  • Networks: Communication between containers

1.3 Current Trends: Shift-Left Security

What is Shift-Left Security?

Moving security practices earlier in the development lifecycle rather than treating it as a final gate.

Key Practices:

  • Early Integration: Security from the first line of code
  • CI/CD Security Scanning: Automated vulnerability detection in pipelines
  • Developer-Centric Tools: IDE plugins, pre-commit hooks
  • Policy-as-Code: Define security policies in version control
  • Continuous Monitoring: Real-time threat detection

Benefits: Catch vulnerabilities early, reduce remediation costs, faster releases

Docker Hardened Images (DHI)

The new standard for secure container images - Available FREE under Apache 2.0 license

Key Features:

  • Near-Zero CVEs: Minimal attack surface with continuous patching
  • SLSA Level 3 Provenance: Verifiable build integrity

  • Drop-in Adoption: Compatible with Alpine and Debian variants

Enterprise Options

  • FIPS/STIG Compliance: Government and regulated industry requirements
  • 7-Day CVE Patches: Critical vulnerability fixes in under a week
  • Extended Lifecycle Support (ELS): +5 years of security updates

Supply Chain Security

Why Supply Chain Security Matters

Containers can introduce vulnerabilities through dependencies, base images, and build processes. Supply chain attacks target these weak points.

Key Components:

  • SBOM (Software Bill of Materials): Complete inventory of components and dependencies
  • Image Signing & Verification: Cryptographic proof of image origin and integrity
  • Vulnerability Scanning: Automated detection of known CVEs
  • Base Image Selection: Choose minimal, trusted, maintained images
  • Dependency Management: Track and update all dependencies

Best Practices

  • Use official images from trusted sources
  • Implement content trust and image signing
  • Regularly scan images for vulnerabilities
  • Keep base images and dependencies updated

1.4 Container Security Best Practices

üîí Essential Security Practices

  • Minimal Base Images: Use distroless or Alpine images to reduce attack surface
  • Multi-stage Builds: Separate build and runtime environments
  • Run as Non-Root: Use least-privilege principle
  • Secrets Management: Never hardcode credentials, use secret stores
  • Immutable Infrastructure: Treat containers as disposable

Runtime Security

  • Security Scanning: Continuous vulnerability assessment
  • Runtime Monitoring: Detect anomalous behavior
  • Network Segmentation: Isolate containers with network policies
  • Resource Limits: Prevent resource exhaustion attacks
  • Regular Updates: Keep images and dependencies current
