Application Insights for .NET

Application Performance Monitoring

Key Capabilities:

  • Auto-instrumentation: Automatic telemetry collection
  • Request tracking: HTTP requests and dependencies
  • Exception tracking: Unhandled exceptions
  • Performance profiling: Identify bottlenecks
  • Live metrics: Real-time monitoring
  • Availability tests: Synthetic monitoring

Add to .NET Application:

# Install package
dotnet add package Microsoft.ApplicationInsights.AspNetCore

# Configure in Program.cs
builder.Services.AddApplicationInsightsTelemetry();

Set Connection String:

export APPLICATIONINSIGHTS_CONNECTION_STRING="InstrumentationKey=...;..."

Log Analytics and KQL Queries

Kusto Query Language

Common Query Examples:

// View container logs
ContainerLog
| where TimeGenerated > ago(1h)
| where Name contains "mywebapi"
| project TimeGenerated, LogEntry

// CPU usage by container
Perf
| where ObjectName == "K8SContainer"
| where CounterName == "cpuUsageNanoCores"
| summarize AvgCPU = avg(CounterValue) by bin(TimeGenerated, 5m), InstanceName

// Failed requests
requests
| where success == false
| summarize Count = count() by name, resultCode
| order by Count desc

Alerts and Dashboards

Proactive Monitoring

Create Metric Alert:

az monitor metrics alert create \
  --name high-cpu-alert \
  --resource-group rg-containers \
  --scopes  \
  --condition "avg Percentage CPU > 80" \
  --window-size 5m \
  --evaluation-frequency 1m \
  --action

Create Log Alert:

az monitor log-analytics query \
  --workspace  \
  --analytics-query "requests | where success == false | count"

Alert Best Practices:

  • Set appropriate thresholds to avoid alert fatigue
  • Use action groups for notifications (email, SMS, webhook)
  • Implement escalation policies
  • Document runbooks for common alerts

Distributed Tracing with OpenTelemetry

Track Requests Across Services

Why Distributed Tracing?

  • Understand request flow in microservices
  • Identify performance bottlenecks
  • Track dependencies between services
  • Debug complex failures

Add OpenTelemetry to .NET:

# Install packages
dotnet add package OpenTelemetry.Extensions.Hosting
dotnet add package OpenTelemetry.Instrumentation.AspNetCore
dotnet add package OpenTelemetry.Exporter.Console

# Configure in Program.cs
builder.Services.AddOpenTelemetry()
    .WithTracing(tracerProviderBuilder =>
        tracerProviderBuilder
            .AddAspNetCoreInstrumentation()
            .AddHttpClientInstrumentation()
            .AddConsoleExporter());

Local Container Debugging

Debug Before Deploying

View Container Logs:

# View logs from running container
docker logs 

# Follow logs in real-time
docker logs -f 

# View last 100 lines
docker logs --tail 100

Execute Commands in Container:

# Get shell access
docker exec -it  /bin/bash

# Run specific command
docker exec  dotnet --info

# Check processes
docker exec  ps aux

Inspect Container:

# View container details
docker inspect 

# Check networking
docker inspect --format='{{.NetworkSettings.IPAddress}}'

Performance Optimization

Improve Application Performance

Container-Level Optimizations:

  • Right-size resources: Set appropriate CPU/memory limits
  • Use Alpine images: Smaller image = faster pulls
  • Multi-stage builds: Reduce final image size

.NET Optimizations:

  • ReadyToRun (R2R): Pre-compile for faster startup
  • Trimming: Remove unused assemblies
  • Connection pooling: Reuse database connections
  • Async/await: Non-blocking I/O operations

Secrets Management with Azure Key Vault

Secure Credential Storage

Create Key Vault:

az keyvault create \
  --name mykeyvault \
  --resource-group rg-containers \
  --location eastus

Store Secret:

az keyvault secret set \
  --vault-name mykeyvault \
  --name ConnectionString \
  --value "Server=..."

Use in Container Apps:

az containerapp create \
  --name myapi \
  --resource-group rg-containers \
  --environment myenv \
  --image myregistry.azurecr.io/myapi:1.0 \
  --secrets connectionstring=keyvaultref: \
  --env-vars ConnectionStrings__Default=secretref:connectionstring
